gpt-pp-team is an end-to-end protocol replay toolkit for the ChatGPT Team subscription flow. It reverse-engineers the full chain from Stripe Checkout through PayPal billing agreement, ChatGPT manual approval, and Codex OAuth + PKCE, and implements each step as a runnable client. Given a clean proxy and a payment credential, a completed pipeline yields an OAuth `refresh_token` written to `output/results.jsonl`.

## What gpt-pp-team does

The toolkit replays the entire ChatGPT Team subscription protocol — the same sequence a browser performs — as a scriptable, automatable client. From a single command you get account registration (Camoufox + Turnstile), Stripe Checkout replay, PayPal billing agreement authorization, ChatGPT subscription approval, and final Codex OAuth token extraction.

## Three core capabilities

### hCaptcha visual solver

`CTF-pay/hcaptcha_auto_solver.py` is a ~4,000-line standalone hCaptcha solver. It uses a three-layer decision pipeline: a VLM primary path for zero-shot classification, CLIP/OpenCV heuristic fallback covering 12 known hCaptcha challenge types, and a Playwright human-motion executor that synthesizes mouse trajectories and timing to match organic behavior.

### Anti-fraud empirical research

The repository includes real empirical data on ChatGPT's two-layer anti-fraud mechanism: a probe layer that triggers on individual signals and a batch ban layer that retrospectively links registration cohorts. The data covers 45-account batches over 24 hours with an observed ~2% survival rate, plus a corrected model explaining why early assumptions about the mechanism were wrong. See [`docs/anti-fraud-research.md`](https://github.com/DanOps-1/gpt-pp-team/blob/main/docs/anti-fraud-research.md) for the full dataset.

### 12-loop self-healing daemon

`pipeline.py::daemon()` runs continuously without operator intervention. Its 12 recovery loops handle: Webshare API automatic IP rotation, Cloudflare DNS quota cleanup, tmpfs orphan reclamation, gost relay watchdog restart, and DataDome slider auto-drag. The design target is unattended operation over multiple weeks.

## Prerequisites

Before installation, confirm you have all of the following:

- **PayPal account** — a real, loginable PayPal account with email OTP 2FA. You must complete one manual login on first run. You must also disable mobile push notifications in PayPal settings (Settings → Login management → remove mobile device) or automation will stall waiting for an OTP that goes to push instead of email.
- **EU/US proxy** — an egress proxy with an EU or US IP address. PayPal enforces EU-only registration for the Stripe account configuration used here; Stripe enforces country restrictions. SOCKS5 proxies with authentication require a `gost` relay (installed in Step 2 of Installation).
- **Cloudflare zone** — a domain managed in Cloudflare, used to provision catch-all subdomains for registration email addresses. Requires a Cloudflare API token with `Zone:DNS:Edit` and `Zone:Zone:Read` permissions.
- **Linux with Camoufox and Playwright** — a Linux host (Ubuntu 22.04+ or equivalent) capable of running Camoufox (antidetect Firefox) and Playwright. Requires approximately 2 GB RAM and 5 GB disk.
- **Optional: VLM API key** — an OpenAI-compatible VLM endpoint for hCaptcha solving. Residential or residential-like IPs rarely trigger hCaptcha; the solver degrades gracefully to CLIP-only without a VLM key.
- **Optional: CAPTCHA service API key** — a createTask/getTaskResult-compatible CAPTCHA service as a fallback for browser passive captchas.

<Note>
  First-time setup — including tuning the config and completing the PayPal manual login — typically takes 1–3 hours. Once the daemon is running stably, each pipeline cycle completes in approximately 5 minutes.
</Note>
